# L-Agent 日志系统设计方案

> 状态: 待 Review | 日期: 2026-07-11

## 1. 现状

项目目前没有日志系统。唯一与"日志"相关的代码是两个空的 middleware 占位符：

- [agent/middleware/model.py](../agent/middleware/model.py) — `TraceRecord`，仅 `return next_call()`
- [agent/middleware/tool.py](../agent/middleware/tool.py) — `AuditRecord`，仅 `return next_call()`

这两个类说明日志插入点早已被预留在 middleware 链中，只是尚未实现。

## 2. 设计目标

1. **以 session_id 为粒度**：每个 session 一个日志文件，存放于 `workspace/logs/`
2. **结构化日志**：JSON Lines 格式，方便 grep / jq / 脚本分析
3. **完整记录**：参数、返回值、异常信息都完整记录，不做截断或脱敏
4. **只写文件**：日志不通过 event stream 推给 UI，不干扰 Rich Console 渲染
5. **不污染业务代码**：利用现有的 Runner + Middleware 架构注入日志，业务 step 无需改动

## 3. 输出格式

### 3.1 文件路径

```
workspace/logs/
├── sess_<session_id>.jsonl
├── sess_<session_id>.jsonl
└── ...
```

### 3.2 JSON Lines 通用字段

每条日志自动携带：

```json
{
  "ts":     "2026-07-11T06:30:01.123Z",   // UTC ISO 8601 时间戳
  "event":  "model.done",                  // 事件类型 (见 §4)
  // ... 事件特定字段
}
```

`ts` 由 `AgentLogger.log()` 自动生成，调用方无需传入。

## 4. 日志事件类型

```json

  "event": "run.start",
  "session_id": "sess_abc",
  "branch_id": "branch_01",
  "run_id": "r1",
  "input": "用户输入原文"
}
```

**插入点**: Runner `_react_loop` — yield ToolStart 之前（streaming 路径）+ Middleware `AuditRecord`（非 streaming 路径）

```json
  "tool_name": "read_file",
  "arguments": {
    "path": "/foo/bar.txt"
  }

```

### 4.7 `tool.done`

**插入点**: Runner `_react_loop` — yield ToolDone 之前（streaming 路径）+ Middleware `AuditRecord`（非 streaming 路径）

```json
{
  "event": "tool.done",
  "run_id": "r1",
  "tool_name": "read_file",
  "elapsed_ms": 45,
  "status": "ok",
  "result": "file content here..."
}
```

`status` 枚举：`"ok"` | `"denied"` | `"error"`

## 5. 技术方案

### 5.1 新增 `agent/logging/logger.py` — AgentLogger

```python
from __future__ import annotations

import json
import time
from datetime import datetime, timezone
from pathlib import Path
from typing import Any


class AgentLogger:
    """以 session_id 为文件名，写 JSON Lines 到 logs_dir。

    用法:
        logger = AgentLogger(Path("workspace/logs"), "sess_abc")
        logger.log(event="run.start", run_id="r1", input="hello")
    """

    def __init__(self, logs_dir: Path, session_id: str) -> None:
        self._file = logs_dir / f"{session_id}.jsonl"
        self._file.parent.mkdir(parents=True, exist_ok=True)

    def log(self, **kwargs: Any) -> None:
        entry: dict[str, Any] = {
            "ts": datetime.now(timezone.utc).isoformat(),
            **kwargs,
        }
        with open(self._file, "a", encoding="utf-8") as f:
            f.write(json.dumps(entry, ensure_ascii=False, default=str) + "\n")
```

设计要点：

- **追加写、行缓冲**：每条 `log()` 立即 `flush` 到磁盘（`open("a")` + 写完即关），避免进程崩溃丢日志。Agent 场景日志量不大（每秒至多数条），IO 开销可忽略。
- **`default=str`**：对无法 JSON 序列化的对象（Exception、dataclass 等）做 fallback，确保日志写入不抛异常。
- **线程安全**：当前架构是单线程 async，无需加锁。未来如需多线程，可改为 `logging` 模块的 `QueueHandler`。

### 5.3 Middleware 填充实现

#### TraceRecord（model 中间件）

```python
# agent/middleware/model.py — TraceRecord

class TraceRecord(Middleware):
    def __init__(self) -> None:
        super().__init__("trace.record", ActionName.model_call)

    def __call__(self, ctx: RunContext, next_call: Callable[[], Any]) -> Any:
        if ctx.logger is None:
            return next_call()

        t0 = time.time()
        # model.start
        ctx.logger.log(
            event="model.start",
            run_id=ctx.run_id,
            iteration=ctx.budget.consumed_iterations,
            messages_count=len(ctx.current_model_request.messages) if ctx.current_model_request else 0,
            tools_count=len(ctx.current_model_request.tools) if ctx.current_model_request else 0,
        )

        result = next_call()

        elapsed_ms = (time.time() - t0) * 1000
        # model.done
        response = result if isinstance(result, ModelResponse) else None
        ctx.logger.log(
            event="model.done",
            run_id=ctx.run_id,
            iteration=ctx.budget.consumed_iterations,
            elapsed_ms=round(elapsed_ms, 1),
            tokens_in=response.usage.input_tokens if response else 0,
            tokens_out=response.usage.output_tokens if response else 0,
            finish_reason=response.finish_reason if response else "",
            content=response.content if response else str(result),
            reasoning_content=response.reasoning_content if response else "",
            tool_calls=[{"id": tc.id, "name": tc.name, "arguments": tc.arguments}
                        for tc in (response.tool_calls if response else [])],
        )
        return result
```

#### AuditRecord（tool 中间件）

```python
# agent/middleware/tool.py — AuditRecord

class AuditRecord(Middleware):
    def __init__(self) -> None:
        super().__init__("audit.record", ActionName.tool_call)

    def __call__(self, ctx: RunContext, next_call: Callable[[], Any]) -> Any:
        if ctx.logger is None:
            return next_call()

        t0 = time.time()
        # tool.start — 从 context 中取当前 tool plan 信息
        tool_calls = []
        plan = ctx.current_tool_plan
        if plan and hasattr(plan, "calls"):
            tool_calls = plan.calls

        for tc in tool_calls:
            ctx.logger.log(
                event="tool.start",
                run_id=ctx.run_id,
                tool_name=tc.tool_name,
                arguments=tc.arguments if hasattr(tc, "arguments") else {},
            )

        result = next_call()

        elapsed_ms = (time.time() - t0) * 1000
        # tool.done
        if isinstance(result, list):
            for r in result:
                if isinstance(r, ToolResult):
                    ctx.logger.log(
                        event="tool.done",
                        run_id=ctx.run_id,
                        tool_name=getattr(r, "call_id", "") or r.tool_name if hasattr(r, "tool_name") else "",
                        elapsed_ms=round(elapsed_ms, 1),
                        status=r.status.value if hasattr(r, "status") else "ok",
                        result=getattr(r, "content", str(r)),
                    )
        return result
```

## 7. 设计决策记录

| 决策 | 结论 | 原因 |
|------|------|------|
| 日志文件名 | `{session_id}.jsonl` | session 是用户交互的自然边界，一个 session 就是一次 CLI 会话 |
| 存储位置 | `workspace/logs/` | 与其他 workspace 数据（timeline.db）保持一致 |
| 日志格式 | JSON Lines | 易于 grep/jq，也易于未来导入 ELK/Databend 等分析工具 |
| 脱敏处理 | 不做 | workspace 目录仅在本地，不会上传 |
| 内容截断 | 不截断，完整记录 | 调试时完整上下文至关重要 |
| 输出目标 | 仅文件 | 不影响终端 UI（Rich Console 渲染） |
| 配置化 | 暂不做 | 保持简单，后续可加 `settings.logging` |
| 日志轮转 | 暂不做 | 单 session 日志量有限，后续可加 `RotatingFileHandler` |

## 8. 后续可扩展方向

- **日志级别过滤**：增加 `settings.logging.level` 配置，控制只记录 INFO+ 或 DEBUG
- **日志轮转**：大 session 场景下按大小自动切分
- **性能分析**：从 JSONL 中提取 model/tool 耗时做统计分析
- **远程上报**：增加 OTLP / Sentry handler 做线上监控
- **replay 支持**：日志中记录足够信息后可回放 session 用于调试复现
