# 实施流程 SOP

## 流程步骤

### 1. Review 本轮计划文档

- 阅读当前迭代的计划文档，明确目标和范围
- 确认需求细节、优先级和验收标准
- 标记疑问点，必要时与用户对齐

### 2. 了解目前代码现状

- 查看相关模块的代码结构和实现逻辑
- 阅读已有测试用例，理解当前行为
- 检查 git 历史，了解近期变更上下文

### 3. 实施推进

- 按计划文档逐步实现功能或修复
- 遵循项目编码规范和架构约定
- 保持小步提交，每个变更聚焦单一职责

### 4. 测试验收

- 运行已有测试套件，确保无回归
- 针对新增功能编写对应测试用例
- 验证边界条件和异常路径

### 5. 更新进度

- 标记已完成的任务项
- 记录遇到的问题和解决方案
- 同步阻塞项或风险点

### 6. 汇报总结

- 概述本轮完成的工作内容
- 说明与计划的偏差及原因
- 提出下一步建议

### 7. 提交代码

- 确认所有测试通过
- 编写清晰的 commit message
- 请示用户是否提交并推送代码
